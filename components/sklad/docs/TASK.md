# TASK

Статус: сырой scope из чата, без исследования и без декомпозиции по реализации.

Рабочее название модуля: `Sklad`

Возможное бизнес-название:

- `Склад`
- при необходимости позже можно переименовать во что-то точнее

## 1. Объединение модулей

Нужно объединить в один новый модуль:

- `sklad_items_module_new` — новые товары склада
- `recept_module_new_2` — полуфабрикаты и рецепты
- `site_items_new` — новые товары
- `ed_izmer` — единицы измерения

Текущее указание:

- новые модули не переиспользуем
- код из них копируем
- в новый модуль вставляем отдельно

Отдельно:

- у товаров склада и рецептов есть свои категории
- их можно объединить в одну сущность: `Категории склада`
- внутри показывать, сколько товаров / рецептов / заготовок используется в этой категории

Также:

- перенести права доступа на новый модуль

## 2. Перенос аллергенов и состава

Нужно:

- аллергены перенести из заготовок в товар
- состав перенести из заготовок в товар
- возможные аллергены перенести из заготовок в товар

Также нужно перенести:

- права доступа
- поля
- данные, которые уже были заполнены

## 3. Товары сайта

Для товаров сайта:

- сейчас указываем БЖУ
- рядом надо добавить поле, которое относительно введенного БЖУ показывает калорийность
- калорийность считается по формуле
- формула, вероятно, уже где-то есть
- возможно, она уже используется при сохранении в этом модуле

Также:

- посмотреть, почему не отображаются картинки

## 4. Длинные названия инпутов

При создании / редактировании есть инпуты с длинным названием.

Нужно попробовать один из вариантов:

- сделать название в несколько строк
- либо оставить короткое название, а полное описание вынести в tooltip

## 5. Поля действия

Во всех указанных модулях склада есть поле `Действует с`.

Нужно добавить поле:

- `Действует до`

## 6. Удаление позиций

Во все указанные модули склада нужно добавить кнопку `Удалить`.

Серверная логика:

- проверять, что позиция прямо сейчас нигде не участвует
- проверять, что позиция никогда не участвовала
- только тогда разрешать удаление
- если где-то используется или использовалась, показывать, что удаление запрещено

Варианты реализации:

- проверка при удалении
- либо вычислять флаг при открытии позиции

Также:

- на удаление нужны отдельные права доступа

## 7. История изменений

Нужно полностью переделать ведение истории в модулях склада так, чтобы можно было полностью посмотреть состояние.

Отдельно для товаров сайта:

- сделать хранение истории картинок
- сейчас картинки просто перезаписываются

Если понадобится:

- можно создать отдельный bucket под хранение истории картинок

Примечание:

- в исходном чате к этому пункту был приложен скрин `image.png`
- файл сохранен в модуле: `image.png`

## 8. Активные и архивные позиции

Все позиции можно разделить на:

- активные
- архивные (без активности)

Отдельное указание:

- чтобы не плодить категории, сделать архив

## 9. Scope "все модули склада"

Под `все модули склада` в рамках этой задачи понимаются только:

- товары сайта
- товары склада
- рецепты
- заготовки
