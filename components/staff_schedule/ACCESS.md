# Карта доступов staff_schedule

Модуль использует legacy-поле `access` из `get_all` / `get_graph`.

`*_view` показывает данные, `*_edit` разрешает точечное редактирование, `*_access` дает доступ к действию. `access` не наследуется от `view` или `edit`.

## Ограничения не из access

- Поля рабочего времени за день ограничиваются датой, `kind` и `check_period`.
- Будущие даты для рабочего времени должны быть доступны для редактирования; поля здоровья доступны только за текущий день.
- Месячное редактирование прошлых месяцев запрещено всем, кроме `MEGA`.
- Обжалование ошибок использует backend-поле `is_edit` из карточки ошибки.
- `downloadHJ` не должен скрываться из-за отсутствия отдельного export-key: в FE API для него нет отдельного access-флага.
- Изменение уровня директора сейчас открывается по условию `graphKind !== "other"` и не имеет отдельного правила доступа.

## Dev-инструмент

`StaffScheduleAccessTester` доступен только в `development`. Он позволяет добавить отсутствующие ключи staff_schedule, применить пресеты доступов, локально переопределить `kind` и проверить итоговую module-policy поверх сырых ключей.
